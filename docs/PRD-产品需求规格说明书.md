## 1. 项目概述

### 1.1 项目背景

#### 1.1.1 现状问题

当前课题组在学术资源管理和组内交流方面存在以下痛点：

| 问题类别               | 具体描述                                                             |
| ---------------------- | -------------------------------------------------------------------- |
| **论文管理分散** | 所有论文最终版存储在导师处，学生需要点对点传论文，无法自主获取和查阅 |
| **知识传承断层** | 前人研究经验和遇到的问题难以传承，后人可能重复踩坑                   |
| **交流渠道匮乏** | 组内交流依赖微信群等即时通讯工具，有价值的信息难以沉淀和检索         |
| **资源获取低效** | 模板、表格等官方资源分散存储，版本管理混乱                           |

#### 1.1.2 项目目标

建立课题组内部的AI学术交流平台，实现：

1. **论文集中管理**：统一存储、分类、检索课题组所有论文资源
2. **知识沉淀传承**：通过经验贴、下载中心等功能沉淀组内知识资产
3. **高效学术交流**：提供评论、收藏、点赞等互动功能促进组内交流
4. **智能辅助检索**：集成AI助手，基于RAG技术实现智能问答和知识推荐

### 1.2 核心价值

| 价值维度           | 描述                                           |
| ------------------ | ---------------------------------------------- |
| **效率提升** | 学生可自主检索和下载论文，减少导师分发负担     |
| **知识复用** | 经验贴沉淀问题解决方案，避免重复踩坑           |
| **协作增强** | 评论、收藏等功能促进组内学术讨论               |
| **智能辅助** | AI助手基于知识库提供智能问答，提升信息获取效率 |

### 1.3 目标用户画像

#### 1.3.1 学生用户

| 属性               | 描述                                   |
| ------------------ | -------------------------------------- |
| **角色定位** | 普通用户，内容消费者和贡献者           |
| **典型年龄** | 20-30岁                                |
| **技术能力** | 基本计算机操作能力，熟悉学术论文阅读   |
| **核心需求** | 查阅论文、分享经验、获取资源、学术交流 |
| **使用频率** | 高频（每周多次）                       |
| **权限范围** | 发布经验贴、评论、点赞、收藏、下载论文 |

#### 1.3.2 教师用户

| 属性               | 描述                                                          |
| ------------------ | ------------------------------------------------------------- |
| **角色定位** | 管理员，内容审核者和资源发布者                                |

| **技术能力** | 基本计算机操作能力                                            |

| ------------------------ | --------- | -------------------- |
| **当前用户数**     | 约10-15人 | 课题组在读成员       |
| **预计峰值用户数** | 300人     | 考虑毕业用户账户保留 |
| **并发用户数**     | 约30人    | 峰值时段估计         |
| **扩展预留**       | 支持      | 预留多课题组扩展接口 |

### 1.5 核心应用场景

#### 场景1：论文查阅与下载

```
用户：研一学生小李
场景：小李正在进行文献调研，需要查阅课题组前人的研究成果
流程：
1. 登录系统，进入论文页面
2. 通过关键词搜索相关论文
3. 点击论文标题查看详情（作者、摘要、关键词等）
4. 下载PDF原文进行深入阅读
```

#### 场景2：经验分享与交流

```
用户：研三学生小王
场景：小王在实验中遇到了一个棘手问题并成功解决，希望记录下来帮助后来者
流程：
1. 进入经验贴页面，选择"科研技巧"板块
2. 点击发布，使用富文本编辑器撰写经验贴
3. 上传相关附件（代码、数据样例等）
4. 发布后，其他同学可以评论、点赞、收藏
```

#### 场景3：AI智能问答

```
用户：研二学生小张
场景：小张对某个研究方向有疑问，希望快速获取相关信息
流程：
1. 点击右侧AI助手按钮，打开侧边栏
2. 输入问题："课题组在深度学习方向有哪些研究成果？"
3. AI助手基于知识库检索相关论文和经验贴
4. 返回回答并附上3篇最相关的文献链接
```

### 2.3 用户管理模块

#### 2.3.1 用户注册

| 属性               | 描述                       |
| ------------------ | -------------------------- |
| **功能编号** | UM-001                     |
| **功能名称** | 学生用户注册               |
| **优先级**   | P0                         |
| **触发条件** | 用户在登录页点击"注册"按钮 |

**处理流程**：

```
开始
  ↓
用户点击"注册"按钮
  ↓
进入注册页面
  ↓
用户填写注册信息：
  - 姓名（必填，2-20字符）
  - 学号（必填，唯一性校验，格式：数字）
  - 年级（必填，格式：XXXX级，如2023级）
  - 密码（必填，6-20字符，需包含字母和数字）
  - 确认密码（必填，需与密码一致）
  ↓
前端表单校验
  ↓ [校验失败]
提示错误信息，用户修正
  ↓ [校验成功]
提交注册请求
  ↓
后端校验学号唯一性
  ↓ [学号已存在]
返回错误提示"学号已注册"
  ↓ [学号可用]
创建用户记录（默认密码加密存储，角色为学生）
  ↓
返回成功提示
  ↓
跳转至登录页面
  ↓
结束
```

**预期结果**：

- 注册成功后用户记录存入数据库
- 用户角色默认为"学生"
- 用户被引导至登录页面

**异常处理**：

| 异常场景   | 处理方式                       |
| ---------- | ------------------------------ |
| 学号已存在 | 提示"该学号已注册，请直接登录" |
| 网络错误   | 提示"网络异常，请稍后重试"     |
| 必填项为空 | 前端提示"请填写完整信息"       |

#### 2.3.2 用户登录

| 属性               | 描述                       |
| ------------------ | -------------------------- |
| **功能编号** | UM-002                     |
| **功能名称** | 用户登录                   |
| **优先级**   | P0                         |
| **触发条件** | 用户在登录页点击"登录"按钮 |

**处理流程**：

```
开始
  ↓
用户进入登录页面
  ↓
用户输入登录信息：
  - 账号（支持：管理员ID / 姓名 / 学号）
  - 密码
  - 记住用户名密码（可选）
  - 7天免登录（可选）
  ↓
前端表单校验
  ↓ [校验失败]
提示错误信息
  ↓ [校验成功]
提交登录请求
  ↓
后端验证账号密码
  ↓ [验证失败]
提示"账号或密码错误"
  ↓ [验证成功]
生成JWT Token
设置Token过期时间：
  - 勾选7天免登录：7天
  - 未勾选：24小时
  ↓
返回用户信息和Token
  ↓
前端存储登录状态：
  - 勾选记住用户名密码：存储至localStorage
  - Token存储至cookie/sessionStorage
  ↓
跳转至首页
  ↓
结束
```

**预期结果**：

- 登录成功后用户获得访问权限
- Token存储在客户端，用于后续请求认证
- 根据用户角色显示不同的导航菜单

**账号识别规则**：

| 输入类型 | 识别规则                       |
| -------- | ------------------------------ |
| 纯数字   | 优先匹配学号，其次匹配管理员ID |
| 中文     | 匹配姓名（教师和学生均可）     |

### 2.4 论文管理模块

#### 2.4.1 论文上传

| 属性               | 描述                           |
| ------------------ | ------------------------------ |
| **功能编号** | PM-001                         |
| **功能名称** | 论文上传                       |
| **优先级**   | P0                             |
| **触发条件** | 用户在论文页点击"上传论文"按钮 |

**支持格式**：

- PDF（推荐）
- DOC/DOCX
- 其他常用文档格式

**处理流程**：

```
开始
  ↓
用户点击"上传论文"按钮
  ↓
弹出文件选择框（支持多选）
  ↓
用户选择一个或多个文件
  ↓
前端校验文件格式和大小（单文件≤50MB）
  ↓ [校验失败]
提示错误信息
  ↓ [校验成功]
显示上传进度
  ↓
文件上传至服务器临时目录
  ↓
后端进行论文解析（异步处理）
  ↓
显示论文列表（本次上传可见）：
  - 文件名
  - 解析状态（解析中/解析成功/解析失败）
  - 论文信息按钮
  - 删除按钮
  ↓
用户可操作：
  - 点击"论文信息"：查看/编辑解析结果
  - 点击"删除"：从列表中移除
  - 点击"继续上传"：继续添加文件
  - 点击"确认发布"：提交所有论文
  ↓
用户点击"确认发布"
  ↓
保存论文信息至数据库
文件转存至永久存储目录
  ↓
提示"上传成功"
  ↓
跳转至论文详情页或论文列表页
  ↓
结束
```

**论文信息字段**：

| 字段名   | 类型     | 必填 | 来源                                   |
| -------- | -------- | ---- | -------------------------------------- |
| 标题     | String   | 是   | 自动解析/手动填写                      |
| 作者     | String   | 是   | 自动解析/手动填写                      |
| 摘要     | Text     | 否   | 自动解析/手动填写                      |
| 关键词   | String   | 否   | 自动解析/手动填写                      |
| 论文类型 | Enum     | 是   | 自动识别/手动选择（期刊论文/学位论文） |
| DOI号    | String   | 否   | 自动解析/手动填写（仅期刊论文）        |
| 文件路径 | String   | 是   | 系统生成                               |
| 上传者ID | Long     | 是   | 系统获取                               |
| 上传时间 | DateTime | 是   | 系统生成                               |

**异常处理**：

| 异常场景           | 处理方式                         |
| ------------------ | -------------------------------- |
| 文件格式不支持     | 提示"不支持的文件格式"           |
| 文件过大           | 提示"文件大小超过50MB限制"       |
| 解析失败           | 显示"解析失败"，允许手动填写     |
| 用户未确认关闭页面 | 提示"有未发布的论文，确认离开？" |

#### 2.4.2 论文解析

| 属性               | 描述                       |
| ------------------ | -------------------------- |
| **功能编号** | PM-002                     |
| **功能名称** | 论文自动解析               |
| **优先级**   | P0                         |
| **触发条件** | 论文文件上传成功后自动触发 |

**解析流程**：

```
开始
  ↓
接收论文文件
  ↓
文件格式转换（非PDF转为PDF）
  ↓
提取文本内容
  ↓
调用解析服务提取元数据：
  - 标题
  - 作者
  - 摘要
  - 关键词
  - DOI号（如有）
  ↓
判断论文类型：
  - 检测是否有DOI → 期刊论文
  - 检测是否有学位论文特征 → 学位论文
  ↓
返回解析结果
  ↓ [解析成功]
显示解析结果供用户确认/编辑
  ↓ [解析失败]
标记为"解析失败"，允许手动填写
  ↓
结束
```

**解析策略**：

| 策略          | 说明                               | 优先级 |
| ------------- | ---------------------------------- | ------ |
| PDF元数据提取 | 从PDF文件属性中提取标题、作者      | 高     |
| 文本规则匹配  | 从正文开头匹配标题、作者、摘要等   | 高     |
| OCR识别       | 对扫描版PDF进行OCR识别（可选实现） | 低     |

| **优先级**   | P2     |

**对话流程**：

```
开始
  ↓
用户输入问题
  ↓
后端接收问题
  ↓
调用智谱AI API
  ↓
RAG检索知识库（论文、经验贴、下载中心内容）
  ↓
构建Prompt（问题 + 相关知识库内容）
  ↓
获取AI回答
  ↓
检索最相关的3篇引用文章
  ↓
返回回答 + 引用链接
  ↓
前端显示回答内容
回答底部显示引用链接（可点击跳转）
  ↓
保存对话记录
  ↓
结束
```

**引用链接格式**：

- 论文：显示论文标题，点击跳转论文详情页
- 经验贴：显示经验贴标题，点击跳转经验贴详情页
