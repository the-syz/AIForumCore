## 2. 服务器购买指南

### 2.1 阿里云试用服务器申请

1. 登录 [阿里云官网](https://www.aliyun.com/)
2. 进入「试用中心」或搜索「云服务器试用」
3. 选择配置：
   - **规格**：2核2GB（101天）
   - **镜像**：阿里云Linux 3
   - **预装应用**：Docker

### 2.2 推荐配置选择

| 配置项 | 推荐选择 | 说明 |
|--------|---------|------|
| **规格** | 2核2GB 101天 | 试用期更长 |
| **镜像** | 阿里云Linux 3 | 兼容CentOS，轻量高效 |
| **预装应用** | Docker | 开箱即用，省去安装步骤 |
| **地域** | 华东/华北 | 选择离用户近的地域 |

### 2.3 安全组配置

购买后需要配置安全组，开放以下端口：

| 端口 | 协议 | 用途 | 来源 |
|------|------|------|------|
| 22 | TCP | SSH登录 | 0.0.0.0/0 或 你的IP |
| 80 | TCP | HTTP访问 | 0.0.0.0/0 |
| 443 | TCP | HTTPS访问 | 0.0.0.0/0 |

**配置步骤**：
1. 进入ECS控制台 → 实例列表
2. 点击实例ID进入详情页
3. 左侧菜单点击「安全组」
4. 点击「配置规则」→「入方向」→「手动添加」
5. 添加上述端口规则

## 3. 环境准备

### 3.1 连接服务器

#### Windows用户连接方式

**方式一：阿里云控制台远程连接**
1. 登录阿里云控制台
2. 进入ECS实例列表
3. 点击「远程连接」→「通过Workbench远程连接」

**方式二：使用SSH工具**

推荐工具：
- **MobaXterm**（推荐，功能全面）
- **Xshell**
- **Windows Terminal + OpenSSH**

连接命令：
```bash
ssh root@你的服务器公网IP
# 首次连接输入 yes，然后输入密码
```

### 3.2 验证Docker环境（预装Docker可跳过安装）

```bash
# 检查Docker版本
docker --version

# 检查Docker Compose版本
docker compose version

# 如果显示版本号，说明Docker已安装
```

**如果Docker未安装**，执行以下命令：
```bash
# 安装Docker
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动Docker
sudo systemctl start docker
sudo systemctl enable docker
```

### 3.3 配置Docker镜像加速（国内必做）

```bash
# 创建配置目录
sudo mkdir -p /etc/docker

# 配置镜像加速
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://docker.1panel.live",
    "https://hub.rat.dev",
    "https://docker.chenby.cn",
    "https://docker.m.daocloud.io"
  ],
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
EOF

# 重启Docker
sudo systemctl daemon-reload
sudo systemctl restart docker
```

### 3.4 增加Swap空间（2GB内存必做）

**什么是Swap？**

Swap（交换空间）是把硬盘空间当作内存使用的技术。服务器只有2GB物理内存，运行MySQL、Redis、后端等多个容器可能会内存不足，创建Swap可以：
- 防止内存不足导致服务崩溃
- 提供额外的"虚拟内存"
- 相当于给服务器增加了2GB备用内存

**配置步骤**：

```bash
# 1. 创建一个2GB大小的文件
sudo fallocate -l 2G /swapfile

# 2. 设置文件权限（仅root可读写，保证安全）
sudo chmod 600 /swapfile

# 3. 将文件格式化为swap格式
sudo mkswap /swapfile

# 4. 启用swap（立即生效）
sudo swapon /swapfile

# 5. 写入配置文件（重启后自动启用）
echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab

# 6. 验证是否生效（应该看到Swap一栏有2GB）
free -h
```

**验证成功示例**：
```
              total        used        free      shared  buff/cache   available
Mem:          1.9Gi       500Mi       1.2Gi        10Mi       200Mi       1.3Gi
Swap:         2.0Gi          0B       2.0Gi
```

## 4. 代码部署

### 4.1 方式一：Git部署（推荐）

**优点**：更新方便、版本可回退、迁移简单

#### 步骤1：推送代码到Git仓库

**在本地Windows执行**：

```bash
# 进入项目目录
cd F:\AIForumCore

# 初始化Git（如果还没有）
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit: AIForum project"

# 添加远程仓库（替换为你的仓库地址）
# GitHub私有仓库
git remote add origin https://github.com/你的用户名/AIForumCore.git

# 或 Gitee私有仓库（国内访问更快）
git remote add origin https://gitee.com/你的用户名/AIForumCore.git

# 推送到远程仓库
git push -u origin main
```

#### 步骤2：在服务器上克隆代码

```bash
# 安装Git
sudo yum install -y git

# 克隆代码（替换为你的仓库地址）
cd /opt
git clone https://github.com/你的用户名/AIForumCore.git aiforum
cd aiforum
```

#### 步骤3：后续更新代码

```bash
# 在服务器上更新代码
cd /opt/aiforum
git pull

# 重新构建并启动
docker compose -f docker-compose.prod.yml --env-file .env.prod up -d --build
```

### 4.2 方式二：本地上传部署

**适用场景**：没有Git仓库，或代码不想公开

#### 使用SCP上传

**在本地Windows PowerShell中执行**：
```powershell
# 上传整个项目到服务器
scp -r F:\AIForumCore root@你的服务器IP:/opt/aiforum
```

#### 使用SFTP工具

使用 FileZilla 或 WinSCP：
1. 新建站点，协议选择SFTP
2. 主机：服务器IP
3. 用户名：root
4. 密码：服务器密码
5. 连接后拖拽上传到 `/opt/aiforum`

### 4.3 配置环境变量

```bash
# 进入项目目录
cd /opt/aiforum

# 复制环境变量模板
cp .env.example .env.prod

# 编辑配置文件
nano .env.prod
```

**填入以下内容**（替换为真实值）：

```bash
# 数据库配置
MYSQL_ROOT_PASSWORD=YourSecureRootPassword123!
MYSQL_USER=aiforum
MYSQL_PASSWORD=YourSecurePassword123!

# 应用配置
# 生成密钥: openssl rand -hex 32
SECRET_KEY=your-secret-key-at-least-32-characters-long

# AI配置
ZHIPU_API_KEY=your_zhipu_api_key_here

# 其他配置
DEBUG=false
```

按 `Ctrl+O` 保存，`Ctrl+X` 退出。

## 5. 启动与验证

### 5.1 构建并启动服务

```bash
# 确保在项目目录
cd /opt/aiforum

# 构建镜像（首次需要10-20分钟）
docker compose -f docker-compose.prod.yml --env-file .env.prod build

# 启动服务
docker compose -f docker-compose.prod.yml --env-file .env.prod up -d

# 查看启动日志
docker compose -f docker-compose.prod.yml logs -f
```

### 5.2 检查服务状态

```bash
# 查看容器状态
docker ps

# 预期输出（所有服务状态为 Up）
CONTAINER ID   IMAGE                    STATUS          PORTS
xxxxx          aiforum-frontend         Up 2 minutes    0.0.0.0:80->80/tcp
xxxxx          aiforum-backend          Up 2 minutes    8000/tcp
xxxxx          aiforum-mysql            Up 2 minutes    3306/tcp
xxxxx          aiforum-redis            Up 2 minutes    6379/tcp
```

### 5.3 初始化管理员

```bash
# 等待MySQL完全启动（约30秒）
sleep 30

# 进入后端容器
docker exec -it aiforum-backend bash

# 创建管理员用户（交互式）
python scripts/init_admin.py

# 或者创建默认管理员（非交互式，工号: admin, 密码: admin123）
# python scripts/init_admin.py --default

# 退出容器
exit
```

**创建管理员时的输入示例**：
```
请输入管理员信息:
------------------------------
姓名: 张老师
工号/学号: T001
密码 (默认为工号): your_password
角色 (1=教师/2=学生管理员，默认1): 1

==================================================
管理员创建成功!
==================================================
  姓名: 张老师
  工号: T001
  角色: 教师
  权限: 管理员
==================================================
```

### 5.4 验证部署

1. **浏览器访问**：`http://你的服务器公网IP`
2. **测试管理员登录**：使用刚创建的管理员账号登录
3. **测试学生注册**：注册一个学生账号测试注册功能

## 6. 部署后配置

### 6.1 配置自动备份

```bash
# 添加执行权限
chmod +x /opt/aiforum/scripts/backup.sh
chmod +x /opt/aiforum/scripts/restore.sh

# 创建备份目录
mkdir -p /opt/backup/aiforum

# 测试备份脚本
/opt/aiforum/scripts/backup.sh

# 配置定时任务（每天凌晨2点执行）
(crontab -l 2>/dev/null; echo "0 2 * * * /opt/aiforum/scripts/backup.sh >> /var/log/aiforum_backup.log 2>&1") | crontab -

# 验证定时任务
crontab -l
```

**备份内容**：
| 备份项 | 说明 |
|--------|------|
| MySQL数据库 | 数据库全量备份 |
| 上传文件 | uploads目录下的所有文件 |
| 向量数据库 | FAISS索引文件 |

**恢复数据**：
```bash
# 查看可用备份
ls -la /opt/backup/aiforum/

# 恢复数据（指定备份日期时间戳）
/opt/aiforum/scripts/restore.sh 20260312_020000
```

## 7. 常见问题

### 7.1 部署问题

#### Q1: Docker镜像拉取失败

**原因**：国内访问Docker Hub较慢

**解决方案**：
```bash
# 重新配置镜像加速
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://docker.1panel.live",
    "https://hub.rat.dev"
  ]
}
EOF
sudo systemctl restart docker
```

#### Q2: 容器启动后立即退出

**排查步骤**：
```bash
# 查看容器日志
docker logs aiforum-backend
docker logs aiforum-mysql

# 查看所有容器状态
docker ps -a
```

#### Q3: 内存不足导致服务崩溃

**解决方案**：
```bash
# 检查内存使用
free -h

# 检查swap是否启用
swapon --show

# 如果swap未启用，重新启用
sudo swapon -a

# 重启服务
docker compose -f docker-compose.prod.yml restart
```

### 7.2 运行问题

#### Q4: 数据库连接失败

**排查步骤**：
```bash
# 检查MySQL容器状态
docker logs aiforum-mysql

# 确保MySQL已完全启动
docker exec aiforum-mysql mysql -u root -p -e "SELECT 1"
```

#### Q5: 文件上传失败

**解决方案**：
```bash
# 检查磁盘空间
df -h

# 检查上传目录权限
docker exec -it aiforum-backend bash
ls -la uploads/
chmod -R 755 uploads/
```

#### Q6: AI对话无响应

**排查步骤**：
```bash
# 检查API Key配置
docker exec -it aiforum-backend bash
echo $ZHIPU_API_KEY

# 测试API连接
curl -s https://open.bigmodel.cn/api/paas/v4/models | head -100
```

### 7.3 维护命令

```bash
# 查看所有容器状态
docker ps

# 查看日志
docker compose -f docker-compose.prod.yml logs -f

# 重启所有服务
docker compose -f docker-compose.prod.yml restart

# 重启单个服务
docker compose -f docker-compose.prod.yml restart backend

# 停止所有服务
docker compose -f docker-compose.prod.yml down

# 更新代码后重新部署
git pull
docker compose -f docker-compose.prod.yml build
docker compose -f docker-compose.prod.yml up -d
```

## 8. 服务器迁移指南

### 8.1 迁移前准备

```bash
# 在旧服务器上打包数据
cd /opt/aiforum

# 执行备份
./scripts/backup.sh

# 打包所有需要迁移的数据
tar -czf /tmp/aiforum_migration.tar.gz \
    /opt/backup/aiforum \
    /opt/aiforum/.env.prod

# 下载到本地（在本地Windows执行）
scp root@旧服务器IP:/tmp/aiforum_migration.tar.gz F:\
```

### 8.2 新服务器部署

```bash
# 1. 克隆代码
cd /opt
git clone https://github.com/你的用户名/AIForumCore.git aiforum
cd aiforum

# 2. 上传迁移数据（从本地上传）
scp F:\aiforum_migration.tar.gz root@新服务器IP:/tmp/

# 3. 解压迁移数据
tar -xzf /tmp/aiforum_migration.tar.gz -C /

# 4. 启动服务
docker compose -f docker-compose.prod.yml --env-file .env.prod up -d --build

# 5. 恢复数据库
./scripts/restore.sh <备份时间戳>
```
