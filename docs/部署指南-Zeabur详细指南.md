## 1. Zeabur 简介

### 1.1 什么是 Zeabur？

Zeabur 是一个现代化的**一键部署平台**，特点如下：

| 特点 | 说明 |
|------|------|
| **开箱即用** | 自动识别项目类型，无需复杂配置 |
| **中文友好** | 完整的中文界面和文档 |
| **按量计费** | 只为实际使用的资源付费 |
| **自动SSL** | 自动配置HTTPS证书 |
| **CI/CD** | 代码推送自动部署 |

### 1.2 定价方案

| 方案 | 价格 | 适合场景 |
|------|------|---------|
| **免费方案** | $0/月 | 静态网站、测试项目 |
| **开发者方案** | $5/月起 | 小型应用、个人项目 |
| **团队方案** | $80/月起 | 团队协作项目 |

**对于AIForum项目**：需要开发者方案（$5/月起），因为包含后端服务和数据库。

### 1.3 Zeabur vs 传统服务器

| 对比项 | Zeabur | 传统服务器 |
|--------|--------|-----------|
| 部署难度 | ⭐ 简单 | ⭐⭐⭐ 中等 |
| 学习成本 | 低 | 高 |
| 配置时间 | 5-10分钟 | 1-2小时 |
| 运维需求 | 几乎无 | 需要 |
| 灵活性 | 中等 | 高 |
| 文件存储 | 需额外配置 | 本地存储 |
| 国内访问 | 海外节点，可能较慢 | 国内节点，速度快 |

## 3. Zeabur 部署步骤

### 3.1 注册并登录

1. 访问 [Zeabur 官网](https://zeabur.com/)
2. 点击右上角「登录」
3. 选择「使用 GitHub 登录」
4. 授权 Zeabur 访问你的 GitHub 仓库

### 3.2 创建项目

1. 点击「新建项目」
2. 选择区域：
   - **AWS 美西**：适合海外用户
   - **GCP 台湾**：适合国内用户（速度较快）
3. 输入项目名称：`aiforum`

### 3.3 添加服务

按照以下顺序添加服务：

#### 步骤 1：添加 MySQL

1. 点击「添加服务」→「预构建」
2. 选择「MySQL」
3. 等待部署完成
4. 记录连接信息（稍后配置环境变量时使用）

#### 步骤 2：添加 Redis

1. 点击「添加服务」→「预构建」
2. 选择「Redis」
3. 等待部署完成

#### 步骤 3：添加后端服务

1. 点击「添加服务」→「Git」
2. 选择你的 GitHub 仓库 `AIForumCore`
3. Zeabur 会自动检测到 `backend` 目录
4. 配置环境变量：

```
DATABASE_URL=${MYSQL_URI}
REDIS_URL=${REDIS_URI}
SECRET_KEY=your-secret-key-at-least-32-characters
ZHIPU_API_KEY=your-zhipu-api-key
DEBUG=false
```

5. 点击「部署」

#### 步骤 4：添加前端服务

1. 点击「添加服务」→「Git」
2. 选择同一个仓库
3. Zeabur 会自动检测到 `frontend` 目录
4. 配置环境变量（如果需要）：
   ```
   VITE_API_BASE_URL=${BACKEND_URL}
   ```
5. 点击「部署」

### 3.4 配置域名

#### 使用 Zeabur 免费域名

1. 进入项目设置
2. 点击「域名」
3. 点击「生成域名」
4. 获得 `xxx.zeabur.app` 格式的域名

#### 使用自定义域名

1. 在域名服务商添加 CNAME 记录
2. 在 Zeabur 中添加自定义域名
3. 等待 SSL 证书自动配置

## 4. 重要配置说明

### 4.1 文件存储问题

⚠️ **重要提示**：Zeabur 容器是无状态的，重启后本地文件会丢失！

**解决方案**：使用 Zeabur 的 Volume 功能

1. 进入后端服务设置
2. 添加持久化存储卷
3. 挂载到 `/app/uploads` 目录

或者使用对象存储服务（如阿里云 OSS、腾讯云 COS）

### 4.2 环境变量配置

在 Zeabur 控制台中配置以下环境变量：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `DATABASE_URL` | 数据库连接 | `${MYSQL_URI}`（自动填充） |
| `REDIS_URL` | Redis连接 | `${REDIS_URI}`（自动填充） |
| `SECRET_KEY` | JWT密钥 | 随机32位字符串 |
| `ZHIPU_API_KEY` | 智谱AI密钥 | 你的API Key |
| `DEBUG` | 调试模式 | `false` |

### 4.3 自动部署配置

Zeabur 支持自动部署：

1. 每次推送到 main 分支，自动触发部署
2. 可在设置中配置部署分支
3. 支持部署前预览

## 6. 常见问题

### Q1: 部署失败，提示构建错误

**解决方案**：
1. 检查 `zbpack.json` 配置是否正确
2. 查看 Zeabur 构建日志
3. 确保 `requirements.txt` 或 `package.json` 存在

### Q2: 数据库连接失败

**解决方案**：
1. 确保使用 `${MYSQL_URI}` 变量
2. 检查 MySQL 服务是否正常运行
3. 查看后端日志排查问题

### Q3: 文件上传后丢失

**解决方案**：
1. 配置持久化存储卷
2. 或使用对象存储服务

### Q4: AI 对话无响应

**解决方案**：
1. 检查 `ZHIPU_API_KEY` 是否正确配置
2. 查看后端日志确认 API 调用情况
