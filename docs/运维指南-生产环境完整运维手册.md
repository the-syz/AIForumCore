## 1. 数据存储位置

### 1.1 Docker Volume 数据位置

#### MySQL 数据库数据
- **Docker Volume 名称**: `mysql-data`
- **宿主机挂载路径**: Docker 自动管理
- **容器内路径**: `/var/lib/mysql`
- **备份挂载**: `./backup` → `/backup` (容器内)

#### Redis 缓存数据
- **Docker Volume 名称**: `redis-data`
- **容器内路径**: `/data`
- **说明**: Redis 数据为临时缓存，无需长期备份

#### 上传文件数据
- **Docker Volume 名称**: `uploads-data`
- **容器内路径**: `/app/uploads`
- **子目录**:
  - `/app/uploads/papers` - 论文文件
  - `/app/uploads/attachments` - 经验贴附件
  - `/app/uploads/editor` - 富文本编辑器图片

#### 向量数据库数据
- **宿主机路径**: `./backend/data`
- **容器内路径**: `/app/data`
- **内容**: FAISS 向量索引文件

### 1.2 宿主机项目目录结构

```
/opt/AIForumCore/
├── .env.prod                    # 生产环境配置文件（敏感！）
├── docker-compose.prod.yml      # 生产环境 Docker Compose 配置
├── backend/
│   └── data/                    # 向量数据库数据
├── backup/                       # 数据库备份目录（MySQL容器挂载）
└── scripts/
    └── backup.sh                # 数据备份脚本
```

## 3. 自动备份配置

### 3.1 备份脚本说明

脚本位置: `scripts/backup.sh`

**功能**:
- 自动备份 MySQL 数据库
- 自动备份上传文件
- 自动备份向量数据库
- 自动清理 7 天前的旧备份
- 记录详细的备份日志

### 3.2 手动执行备份

```bash
cd /opt/AIForumCore

# 赋予执行权限（首次）
chmod +x scripts/backup.sh

# 执行备份
./scripts/backup.sh
```

### 3.3 配置定时自动备份

#### 3.3.1 编辑 crontab

```bash
crontab -e
```

#### 3.3.2 添加定时任务

```bash
# 每天凌晨 2:00 执行备份
0 2 * * * /opt/AIForumCore/scripts/backup.sh >> /var/log/aiforum_backup.log 2>&1

# 每周日凌晨 3:00 清理过期备份（可选，脚本已内置清理）
# 0 3 * * 0 find /opt/backup/aiforum -name "*.gz" -mtime +7 -delete
```

#### 3.3.3 验证定时任务

```bash
# 查看当前定时任务
crontab -l

# 查看备份日志
tail -f /var/log/aiforum_backup.log
```

### 3.4 备份脚本配置

编辑 `scripts/backup.sh` 修改配置：

```bash
# 备份目录
BACKUP_DIR="/opt/backup/aiforum"

# 项目目录
PROJECT_DIR="/opt/AIForumCore"

# 环境变量文件
ENV_FILE="${PROJECT_DIR}/.env.prod"

# 保留天数
RETENTION_DAYS=7
```

## 4. 数据恢复流程

### 4.1 恢复 MySQL 数据库

#### 步骤 1: 停止后端服务

```bash
cd /opt/AIForumCore
docker compose -f docker-compose.prod.yml stop backend
```

#### 步骤 2: 选择备份文件

```bash
# 查看可用的数据库备份
ls -lh /opt/backup/aiforum/db_backup_*.sql.gz

# 选择要恢复的备份文件
BACKUP_FILE="/opt/backup/aiforum/db_backup_20260315_020000.sql.gz"
```

#### 步骤 3: 解压并恢复

```bash
# 解压备份文件
gunzip -c "$BACKUP_FILE" > /tmp/restore.sql

# 读取数据库密码
source /opt/AIForumCore/.env.prod

# 恢复数据库
docker exec -i aiforum-mysql mysql -u root -p"${MYSQL_ROOT_PASSWORD}" aiforum < /tmp/restore.sql

# 清理临时文件
rm /tmp/restore.sql
```

#### 步骤 4: 验证并启动服务

```bash
# 验证数据库
docker exec aiforum-mysql mysql -u root -p"${MYSQL_ROOT_PASSWORD}" -e "USE aiforum; SHOW TABLES;"

# 启动后端服务
docker compose -f docker-compose.prod.yml start backend
```

### 4.2 恢复上传文件

```bash
cd /opt/AIForumCore

# 选择备份文件
BACKUP_FILE="/opt/backup/aiforum/uploads_backup_20260315_020000.tar.gz"

# 停止后端服务
docker compose -f docker-compose.prod.yml stop backend

# 解压到临时目录
mkdir -p /tmp/restore_uploads
tar -xzf "$BACKUP_FILE" -C /tmp/restore_uploads

# 使用 Docker cp 复制到容器
docker cp /tmp/restore_uploads/uploads/. aiforum-backend:/app/uploads/

# 设置正确权限
docker exec aiforum-backend chown -R root:root /app/uploads
docker exec aiforum-backend chmod -R 755 /app/uploads

# 清理临时文件
rm -rf /tmp/restore_uploads

# 启动后端服务
docker compose -f docker-compose.prod.yml start backend
```

### 4.3 恢复向量数据库

```bash
cd /opt/AIForumCore

# 选择备份文件
BACKUP_FILE="/opt/backup/aiforum/vector_backup_20260315_020000.tar.gz"

# 停止后端服务
docker compose -f docker-compose.prod.yml stop backend

# 解压备份
tar -xzf "$BACKUP_FILE" -C /opt/AIForumCore/backend/

# 启动后端服务
docker compose -f docker-compose.prod.yml start backend

# 重新生成向量索引（可选）
# 访问 /api/generate-embeddings 端点（如果有的话）
```

## 6. 监控与日志

### 6.1 Docker 容器日志

#### 查看实时日志

```bash
# 查看所有容器日志
docker compose -f docker-compose.prod.yml logs -f

# 查看特定容器日志
docker logs -f aiforum-backend
docker logs -f aiforum-mysql
docker logs -f aiforum-frontend
```

#### 查看最近日志

```bash
# 查看后端最近 100 行日志
docker logs aiforum-backend --tail 100

# 查看最近 1 小时的日志
docker logs aiforum-backend --since 1h

# 查看特定时间范围的日志
docker logs aiforum-backend --since "2026-03-15T00:00:00" --until "2026-03-15T12:00:00"
```

### 6.2 日志轮转配置

创建 Docker 日志轮转配置 `/etc/docker/daemon.json`:

```json
{
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}
```

重启 Docker 服务:

```bash
systemctl restart docker
```

### 6.3 健康检查

```bash
# 检查后端健康状态
curl http://localhost/health

# 或者通过 Docker 健康检查
docker inspect --format='{{.State.Health.Status}}' aiforum-backend
docker inspect --format='{{.State.Health.Status}}' aiforum-mysql
docker inspect --format='{{.State.Health.Status}}' aiforum-redis
```

## 7. 安全建议

### 7.1 配置文件安全

```bash
# 设置正确的文件权限
cd /opt/AIForumCore
chmod 600 .env.prod
chmod 700 scripts/backup.sh

# 确保只有 root 用户可访问
chown -R root:root /opt/AIForumCore
```

### 7.2 防火墙配置

```bash
# 仅允许必要的端口
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --permanent --add-port=443/tcp
firewall-cmd --permanent --remove-port=3306/tcp
firewall-cmd --permanent --remove-port=6379/tcp
firewall-cmd --permanent --remove-port=8000/tcp
firewall-cmd --reload
```

### 7.3 定期更新

```bash
# 更新系统包
yum update -y  # CentOS/RHEL
apt update && apt upgrade -y  # Ubuntu/Debian

# 更新 Docker 镜像
docker compose -f docker-compose.prod.yml pull
docker compose -f docker-compose.prod.yml up -d
```

### 7.4 访问控制

- 使用强密码（`.env.prod` 中的密码）
- 定期更换数据库密码
- 限制 SSH 访问（使用密钥认证）

## 9. 故障排查

### 9.1 常见问题

#### 问题 1: 容器不断重启

**排查步骤**:
```bash
# 查看容器日志
docker logs aiforum-backend

# 检查容器状态
docker compose -f docker-compose.prod.yml ps

# 检查依赖服务
docker logs aiforum-mysql
docker logs aiforum-redis
```

**常见原因**:
- 数据库连接失败 → 检查 `.env.prod` 配置
- 端口冲突 → 检查宿主机端口占用
- 资源不足 → 检查内存和磁盘空间

#### 问题 2: 数据库连接失败

**排查步骤**:
```bash
# 检查 MySQL 容器状态
docker ps | grep mysql

# 测试数据库连接
docker exec aiforum-mysql mysql -u root -p"${MYSQL_ROOT_PASSWORD}" -e "SELECT 1;"

# 检查数据库配置
source /opt/AIForumCore/.env.prod
echo $DATABASE_URL
```

#### 问题 3: 文件上传失败

**排查步骤**:
```bash
# 检查上传目录权限
docker exec aiforum-backend ls -la /app/uploads/

# 检查磁盘空间
df -h

# 查看后端日志
docker logs aiforum-backend --tail 100

# 检查 Nginx 配置（前端容器）
docker exec aiforum-frontend cat /etc/nginx/conf.d/default.conf
```

### 9.2 紧急恢复流程

#### 完整服务重启

```bash
cd /opt/AIForumCore

# 停止所有服务
docker compose -f docker-compose.prod.yml down

# 等待 10 秒
sleep 10

# 重新启动
docker compose -f docker-compose.prod.yml up -d

# 等待服务启动
sleep 30

# 检查状态
docker compose -f docker-compose.prod.yml ps
```

## 10. 版本更新流程

### 10.1 更新前准备

```bash
# 1. 完整备份
./scripts/backup.sh

# 2. 停止服务
docker compose -f docker-compose.prod.yml down

# 3. 拉取最新代码
git pull origin main
```

### 10.2 执行更新

```bash
cd /opt/AIForumCore

# 重新构建镜像（不使用缓存）
docker compose -f docker-compose.prod.yml build --no-cache

# 启动服务
docker compose -f docker-compose.prod.yml up -d

# 等待服务启动
sleep 30

# 验证服务状态
docker compose -f docker-compose.prod.yml ps
docker logs aiforum-backend --tail 50
```

### 10.3 回滚流程

如果更新出现问题，按以下步骤回滚：

```bash
cd /opt/AIForumCore

# 1. 停止服务
docker compose -f docker-compose.prod.yml down

# 2. 回滚代码
git checkout <previous-commit-hash>

# 3. 恢复数据（如需要）
# 参考第 4 章数据恢复流程

# 4. 使用旧镜像重新构建
docker compose -f docker-compose.prod.yml build --no-cache

# 5. 启动服务
docker compose -f docker-compose.prod.yml up -d
```
